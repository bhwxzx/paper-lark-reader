# 修订阅读笔记

## 触发条件

用户明确要求把追问、启发、补充材料或纠错内容写回论文笔记时使用。

普通论文讨论不得自动进入本流程。

## 前置动作

读取：
- 现有 Markdown 笔记（位于 `<paper_dir>/notes/<paper_name>.paper-note.md`）。
- 本轮需要写入的讨论、候选补充记录、纠错或补充材料。
- `references/note-writing-guide.md`（严格对齐其排版与公式规范）。

## 步骤

1. 解析现有笔记章节，确认是否包含标准结构。
2. 按路由表确定写入位置（**必须精准匹配带数字的现有二级标题**）：

   | 内容类型 | 写入章节 |
   | :--- | :--- |
   | 元数据纠错 | `## 1. 元数据` |
   | 核心概括/开源链接 | `## 2. 核心概括` |
   | 研究问题/背景 | `## 3. 背景与问题` |
   | 方法机制/公式推导 | `## 4. 方法` |
   | 数据构造/仿真环境 | `## 5. 数据与环境` |
   | 实验细节/对比分析 | `## 6. 实验` |
   | 创新、启发、局限、风险 | `## 7. 评价与启发` |

3. 只有完全无法归入上述 7 个标准章节、来自额外材料或需要长篇保留问答上下文时，才在文件末尾创建 `## 8. 补充信息`。
4. **展示修订预览（极其重要）**：写入文件前，必须向用户展示：
   - 将修改的具体章节名称。
   - 每章新增/替换的内容摘要（若是新增，说明是插入到该章末尾还是特定位置）。
   - 是否会改动 `## 1. 元数据`。
5. 只有在用户确认“同意写回”后，才将修订真正写回本地 `<paper_dir>/notes/<paper_name>.paper-note.md` 文件。
6. 写入时必须保持原有的飞书兼容 LaTeX 公式格式（`$math$`）和无序列表风格。
7. 修订后重新执行校验：
   ```bash
   & ".\py.cmd" scripts/paper_lark_cli.py note-check <paper_dir>/notes/<paper_name>.paper-note.md
   ```
8. **云端同步拦截**：若这篇笔记已归档到飞书，用户随后要求“同步到飞书”，必须转入 `kb-upload.md` 走重复检测；同 Title 命中时提示将更新已有云端记录，确认后走覆盖更新分支。

## 验收

- 元数据表通过 `note-check` 校验，未被破坏其 Markdown 表格结构。
- 新内容准确写入了带数字标号的现有章节中，没有生成诸如 `## 方法`（缺少序号）之类的杂乱重复章节。
- `Title` 和 `Abstract` 只有在用户明确纠错时才被修改。
- 写回本地笔记前已展示了修订预览并获得用户确认。
- 未自动上传或同步至云端，除非用户明确下达了上传指令。

## 反例（禁止行为）

- 绝对不要把每次追问都追加成新的杂乱章节。
- 不要在 `## 1. 元数据` 的 `摘要` 字段中混入追问总结，它只能是原文 Abstract 的直译。
- 绝对不要在未获得用户确认预览的情况下，擅自修改本地 `.paper-note.md` 文件。
