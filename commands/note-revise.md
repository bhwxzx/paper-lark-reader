# 修订阅读笔记

## 触发条件

用户明确要求把追问、启发、补充材料或纠错内容写回论文笔记时使用。

普通论文讨论不得自动进入本流程。

## 前置

读取：

- 现有 Markdown 笔记。
- 本轮需要写入的讨论、纠错或补充。
- `references/note-writing-guide.md`。

## 步骤

1. 解析现有笔记章节，确认是否包含标准结构。
2. 按路由表确定写入位置：

   | 内容类型 | 写入章节 |
   |---|---|
   | 元数据纠错 | `## 元数据` |
   | 研究问题 | `## 背景与问题` |
   | 方法机制 | `## 方法` |
   | 数据构造 | `## 数据` |
   | 实验细节 | `## 实验` |
   | 启发、局限、风险 | `## 评价与启发` |

3. 只有无法归入标准章节、来自额外材料或需要保留问答上下文时，才创建 `## 补充信息`。
4. 写入前展示修订预览：列出将修改的章节、每章新增/替换内容摘要，以及是否会改动 `## 元数据`。
5. 只有用户确认后，才将修订写回本地笔记。
6. 修订后重新校验：
   ```bash
   python3 scripts/paper_lark_cli.py note-check <NOTE.md>
   ```
7. 若这篇笔记已归档到飞书，而用户随后要求上传或同步，先走重复检测；同 Title 命中时提示将更新已有云端记录，确认后走上传更新分支。

## 验收

- 元数据表通过 `note-check`。
- 新内容写入明确章节，没有生成杂乱重复章节。
- `Title` 和 `Abstract` 只有在用户明确纠错时才修改。
- 写回本地笔记前已展示修订预览并获得用户确认。
- 不自动上传或同步云端，除非用户明确要求。

## 反例

- 不要把每次追问都追加成新的杂乱章节。
- 不要在 `摘要` 字段加入追问总结。
- 不要改动已确认的 Title/Abstract，除非用户明确纠错。
