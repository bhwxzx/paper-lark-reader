# 智能文献推荐

## 触发条件
当用户要求“推荐论文”、“最近有什么新文章”、“根据我的画像找点文献”、“文献追踪”时触发。

## 前置动作（极度重要）

1. **获取当前系统时间**：执行检索前，请先明确当前的实际年份和月份，以确保“最近半年内”的时间界定是绝对准确的。
2. **获取飞书配置**：读取 `<SKILL_ROOT>/config/defaults.json` 获取 `app_token` 和 `table_id`。
3. **提取已读列表（知识库去重）**：
   为了绝对避免向用户推荐已经阅读过的文献，必须先从飞书 Base 中获取已有论文的标题列表。请静默执行以下命令：
   ```bash
   npx lark-cli base +record-list \
     --base-token <app_token> \
     --table-id <table_id> \
     --field-id Title \
     --limit 500 > .existing_papers.json
   ```
   *读取生成的 `.existing_papers.json`（通常在 `data.items` 数组的 `fields.Title` 中），将其中的标题提取出来作为本次推荐的**黑名单（Blocklist）**。*
4. **优先读取用户画像**：读取本地工作区中的 `研究概述.md`。提取其中的“研究领域关键词”、“关注问题（痛点）”和“方法偏好”，作为推荐的唯一基准。若该文件不存在，可去飞书知识库中线上查找。

## 检索策略（使用联网搜索工具）
请使用你内置的 Web Search（联网搜索）能力，结合具体年份（如 2025/2026），执行 2-3 次不同维度的深度搜索：
- **学术维度**：搜索 arXiv 或 Google Scholar 上与用户画像强相关的最新论文（例如搜索 `"Reinforcement Learning" AND "Sim2Real" 2026 site:arxiv.org`）。
- **前沿解读维度（微信公众号/科技媒体）**：搜索国内优质的学术解读文章，从中找出所提到的论文（例如搜索 `强化学习 机器人运控 最新突破 具身 site:mp.weixin.qq.com` 或相关知乎专栏）。

**🚨 查重过滤**：在拿到搜索结果后，必须与步骤 3 中获取的“已读黑名单”进行交叉比对。如果搜索到的论文标题与黑名单中的标题一致（或高度相似），**必须将其直接丢弃**，另寻其他文献。

## 推荐输出规范
请严格按照以下 Markdown 结构向用户输出推荐报告：

### 🔍 领域微趋势 (Micro-trend)
用 1-2 句话，总结从本次搜索结果中观察到的、与用户方向相关的最新技术趋势。（例如：“最近三个月，大家都在尝试用 Diffusion 模型解决 Sim2Real 的泛化问题...”）

### 📄 专属推荐列表（精选 3-5 篇）
针对每一篇推荐的论文或文章，必须包含以下结构：

#### 1. [中文标题] (原外文标题，若是中文文章则留空)
- **来源与日期**：[例如：arXiv / 微信公众号机器之心/具身 · 最近3个月内]
- **文章链接**：[URL]
- **🔥 画像匹配度**：[例如：95% / ★★★★★]
- **💡 核心亮点**：[用一句话说明这篇文献最大的技术突破或看点]
- **🎯 推荐理由**：[**强制要求**：必须结合 `研究概述.md` 中的具体痛点来写。例如：“您的画像中提到目前面临奖励函数设计困难的痛点，而这篇文章提出了一种自动 Reward Shaping 的新方法，契合需求。”]

### 🤖 下一步操作建议
在列表末尾，向用户提供以下交互选项：
> **需要我为您做进一步处理吗？**
> - 回复 `[序号] + 阅读`：若为 arXiv 论文，我将尝试为您自动下载 PDF 并生成结构化的精读笔记。
> - 回复 `[序号] + 翻译`：我将为您下载并执行全文逐段翻译。
下载pdf需下载到当前工作目录下，并先给用户展示要下载到的目录，用户确认后再下载

### 🧹 清理临时文件
在输出完推荐列表后，请在后台静默执行以下命令，销毁用于查重的临时黑名单，保持工作区干净：
```bash
Remove-Item -Path .existing_papers.json -Force -ErrorAction SilentlyContinue
```

## 反例（禁止行为）
- **【绝对禁止】推荐 `.existing_papers.json` 中已经存在的文献！**
- 绝对不要推荐距离当前系统时间超过 3 年的陈旧论文，除非它是该方向绝对的奠基之作（需注明）。
- 不要推荐与 `研究概述.md` 痛点毫无关联的热门论文。
- 推荐理由绝对不能是泛泛而谈的“这篇文章很有趣”，必须指出它解决了用户画像里的哪个 specific gap。
