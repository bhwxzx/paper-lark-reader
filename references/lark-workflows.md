# 飞书工作流约束

## 固定结构

- 知识库：由配置或用户指定。
- Base：`论文索引`。
- 内部表：`数据表`。
- 默认视图：`概览`。
- 父文档：`论文笔记`。
- 画像文档：`研究概述`。

## 前置检查

除纯论文讨论外，建库后的论文阅读、画像生成、上传归档等依赖库流程开始前必须检查：

- `lark-cli` 是否安装、可运行且当前身份已登录。
- `lark-cli --version` 是否与本文档验证版本 `1.0.19` 一致；不一致时先查 help/schema。
- `<SKILL_ROOT>/config/defaults.json` 是否包含目标知识库所需 token；`<SKILL_ROOT>` 是包含 `SKILL.md` 的 `paper-lark-reader/` 目录。
- 目标知识库、`论文索引`、`数据表`、`论文笔记` 是否可访问。

任一检查失败时，停止当前依赖库流程，不要静默降级为无库流程。

## Base 字段

`概览` 视图可见字段顺序固定：

```text
Title、出版、年份、状态、标签、评分、创建、附件、笔记、Abstract、标题、摘要
```

字段类型要点：

- `Title` 是主列，初始化时只能重命名默认主列，不要删除主列。
- `出版`、`年份` 为单选字段，初始化时不预置选项。
- `状态` 为单选字段，至少包含 `待读`。
- `标签` 为多选字段，初始化时不预置选项。
- `评分` 是 number 字段，UI 样式必须为 Rating。
- `创建` 是系统创建时间字段，不手动写入。
- `附件` 是附件字段，只通过 `+record-upload-attachment` 写入。
- `笔记` 是 URL 样式文本字段，值写作 `[笔记](<论文子文档 wiki URL>)`。
- `Title`、`Abstract`、`标题`、`摘要` 写入 Base 时必须来自 Markdown 笔记 `## 元数据` 表。

字段创建后用 `+view-set-visible-fields` 固定视图顺序；不要依赖 `+field-list` 的返回顺序。

## 建库安全

初始化时必须先按目标名称精确查找知识库。同名知识库可以复用，但不能自动复用。

若查到同名知识库，必须暂停并展示风险：

- 可能误写入旧库或非预期库。
- 可能改变已有 Base、文档、视图或附件结构。
- 可能把历史论文和当前论文混在一起。
- 可能遇到同名但语义不同的 `论文索引` 或 `论文笔记`。

更推荐改用新名称创建干净知识库。只有用户明确选择某个候选并确认风险后，才允许继续扫描该知识库内部结构。

## 风险扫描

建库、复用同名库或上传前，至少检查：

- 当前账号身份、权限和 scope。
- 目标知识库名称、token 或链接是否与用户确认一致。
- `论文索引`、`论文笔记`、`研究概述` 是否存在，以及是否疑似属于本工作流。
- Base 表、字段、字段类型、字段顺序、视图是否符合预期。
- `论文笔记` 父文档是否包含 `<folder_manager>`。
- 是否已有相同 Title、相同 PDF、相同笔记链接或重复 `paper.pdf` 附件。
- 新增选项和破坏性动作是否已单独确认。

## 信息损失防护

任何可能改变、覆盖或移除已有信息的操作，都必须先展示影响范围并获得用户明确确认。适用对象包括本地 Markdown 笔记、飞书文档、Wiki 节点、Base 字段、Base 记录、附件和链接。

必须确认的典型操作：

- 覆盖、更新或重写已有本地笔记。
- `docs +update --command overwrite` 覆盖已有文档正文。
- `drive files patch` 重命名已有文档或资源。
- `base +field-update` 修改已有字段或追加选项。
- `base +record-batch-update` 更新已有记录字段。
- `base +record-upload-attachment` 重新上传或替换已有附件。
- `base +field-delete`、`base +record-delete` 删除已有字段或记录。
- 移动、合并、清理或重构已有知识库资源。

例外：在全新创建的空 Base 初始化过程中，删除系统默认空字段、默认空行和重命名默认主列不视为用户信息损失；复用已有知识库时，同类操作必须单独确认。

## 重复与覆盖

- 本地已有同题笔记时，再次阅读必须展示已有笔记路径和拟修改影响；用户确认后才更新，用户要求保留旧版时才新建副本。
- 上传时若发现同 Title Base 记录、同题子文档或已有笔记链接，必须展示重复项和更新影响；用户确认后才更新已有云端资源。
- 用户明确要求新增时，才新增重复记录或子文档。
- 生成或更新画像时若已存在 `研究概述`，必须确认后才覆盖。
- 讨论论文时不得自动更新本地笔记或云端文档。

## 笔记父文档

`论文笔记` 父文档必须包含飞书子文档列表组件。验收标准是 `docs +fetch --api-version v2 --format pretty` 内容里出现 `<folder_manager>`。

论文子文档创建和写入后，必须确保标题等于论文 Title；若回读仍为 `Untitled`，不得继续后续上传步骤，也不得声称完成。

## 附件

主 PDF 只通过 Base 附件字段上传一次，文件名为 `paper.pdf`。不要同时 Drive 上传一份又 Base 附件上传一份。

## 不做的事

- 不主动删除 Dashboard 或 Workflow。
- 不自动创建出版、年份或标签选项。
- 不在用户未确认时覆盖已有画像、笔记或记录。
