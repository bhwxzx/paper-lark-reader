# 论文笔记写作指南

笔记使用二级标题。默认 7 章，只有追问、额外材料或不能归入前文的信息才创建 `## 补充信息`。
全文排版要求：**多使用无序列表（Bullet points）和文本加粗（Bold）突出重点。**

## 飞书特供公式与排版约束（必须严格遵守）
飞书文档对 Markdown 语法解析有严格要求，遇到核心数学公式时：
1. **行内公式**：必须使用单美元符号，且**美元符号与公式内容之间绝对不能有空格**！正确：`$x=y$`；错误：`$ x=y $`。
2. **块级公式**：必须使用双美元符号，且 `$$` 必须单独占一行。
   正确示范：
   $$
   V(s) = \max_a \left( R(s,a) + \gamma V(s') \right)
   $$
3. **环境限制**：禁止使用 `\begin{equation}` 或复杂的 LaTeX 宏包。如需对齐，只允许使用 `\begin{aligned} ... \end{aligned}` 且必须包裹在 `$$` 内部。

## 1. 元数据

必须输出为一个标准的 Markdown 表格（包含表头 `| 字段 | 内容 |`）。
元数据表只包含以下 8 个字段，不得增加 `状态`、`创建`、`附件`、`笔记`、`翻译` 等自动化字段：

| 字段 | 填写规则 |
| :--- | :--- |
| **Title** | 英文论文标题，规范为 Title Case；不要破坏缩写、模型名、基因/蛋白名或连字符词。 |
| **出版** | 会议、期刊或预印本来源，使用常用简称，不带年份/卷期/页码（如 `ICLR`, `arXiv`）；优先复用 Base 现有选项。 |
| **年份** | 出版年份（如 `2023`）。 |
| **标签** | 2-4 个主题标签（如 `强化学习`, `Sim2Real`），优先复用已有标签。 |
| **评分** | 默认填写 `3`，上传前可由用户修改为 1-5 或留空。 |
| **Abstract** | 英文摘要原文，直接复制，绝对不改写成总结。 |
| **标题** | `Title` 的忠实中文翻译。 |
| **摘要** | `Abstract` 的忠实中文翻译，不能用 AI 总结替代。 |

## 2. 核心概括

用 3-5 句话（建议采用列表形式）说明论文做了什么，覆盖研究问题、核心方法、主要结果、核心结论。
- **开源资源**：如果文中提到了代码库（Github）、项目主页（Project Page）或开源数据集，**必须在这里以超链接形式显著列出**；若无则标明“未提及开源代码”。
- 最后一句固定格式为：**最值得记住的是：** [在此用一句话高度凝练论文的最大价值]。

## 3. 背景与问题

区分并分点列出：
- **领域背景**：研究大方向的现状。
- **已有方法的不足**：前人工作（Baselines）的缺陷。
- **本文要解决的 Gap**：本文提出的具体技术问题。

## 4. 方法

说明整体框架、输入输出、关键模块、训练/推理流程。
- **注意**：不要编造论文没有给出的架构。
- **公式提取**：对于强化学习的状态空间、动作空间、奖励函数（Reward Function）或控制算法的核心推导，必须提取最核心的 1-2 个公式，并严格遵守前文的“飞书特供公式约束”进行书写。

## 5. 数据与环境

针对 AI/控制类论文，独立细读并列出：
- **仿真环境/数据集**：使用了什么物理引擎（如 Isaac Gym, MuJoCo）或基准数据集。
- **数据生成**：数据合成的来源、流程、过滤、打分、去重或域随机化（Domain Randomization）等质量控制手段。
- **sim2real方法**： 若有实物不熟，指出为了将仿真策略迁移到真机上，采用了什么sim2real的具体重点细节方法。

## 6. 实验

按实验目的组织，不按表格流水账复述。每类实验说明：
- 为什么做（验证什么能力）？
- 怎么做（对比了哪些 Baselines）？
- 结果是什么，证明了什么结论？

## 7. 评价与启发

四项分开写，必须带上小标题：
- **创新点**：相比已有工作的具体技术贡献。
- **作者承认的局限**：论文正文或附录中明确提到的不足。
- **阅读者的推断风险**：论文未提及，但在实际落地或复现时你认为可能存在的痛点（如超参数敏感、算力要求高等）。
- **对用户研究方向的启发**：**必须基于当前工作区内的 `研究概述` 文档进行思考**。说明本文的思路如何解决用户画像中的痛点；若画像尚未建立，此项留空。不要在单篇笔记中向用户索要研究方向关键词。
