# 论文笔记写作指南

笔记使用二级标题。默认 7 章，只有追问、额外材料或不能归入前文的信息才创建 `## 补充信息`。

## 元数据

元数据表只包含：

```text
Title、出版、年份、标签、评分、Abstract、标题、摘要
```

不要包含 `状态`、`创建`、`附件`、`笔记`。

字段规则：

- `Title`：英文论文标题，规范为 Title Case；不要破坏缩写、模型名、基因/蛋白名或连字符词。
- `出版`：会议、期刊或预印本来源，使用常用简称，不带年份、卷期、页码或 workshop/session 信息；优先复用 Base 现有出版选项。
- `年份`：出版年份。
- `标签`：2-4 个主题标签，优先复用已有标签。
- `评分`：默认 `3`，上传前可由用户修改为 `1-5` 或留空。
- `Abstract`：英文摘要原文，不改写成总结。
- `标题`：`Title` 的忠实中文翻译。
- `摘要`：`Abstract` 的忠实中文翻译，不能用 AI 总结替代。

## 核心概括

用 3-5 句话说明论文做了什么，覆盖研究问题、核心方法、主要结果、核心结论。最后一句使用：

```text
最值得记住的是：...
```

## 背景与问题

区分：

- 领域背景。
- 已有方法不足。
- 本文具体技术问题和 gap。

## 方法

说明整体框架、输入输出、关键模块、训练/推理流程。不要编造论文没有给出的架构。

## 数据

独立细读训练数据、验证/测试数据、推理数据、外部知识或工具数据。数据合成要写来源、流程、过滤、打分、去重、质量控制和用途。

## 实验

按实验目的组织，不按表格流水账复述。每类实验说明为什么做、怎么做、结果是什么、说明什么。

## 评价与启发

四项分开写：

- **创新点**：相比已有工作的具体技术贡献。
- **作者承认的局限**：论文正文或附录中明确提到的不足。
- **阅读者推断的局限/风险**：论文未提及但值得注意的问题。
- **对用户研究方向的启发**：基于现有 `研究概述` 文档说明可借鉴之处；若画像尚未建立，此项留空。

不要在单篇笔记中重新索要关键词，也不要把跨论文的画像级结论写进单篇笔记。
