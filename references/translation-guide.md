# 论文全文翻译规范

本指南规定了对学术论文进行全文翻译时的语体、术语处理、排版映射及图表引用的严格标准。

## 1. 翻译基本原则
- **学术严谨**：保持客观、正式的学术语气，禁止使用口语化、新闻化或情绪化的表达。
- **忠实原文**：这是“翻译”而非“总结”。必须逐段翻译，不得因为段落过长而擅自删减、合并或用 AI 的话总结。
- **长句拆分**：遇到英文超长复合句时，请在符合中文阅读习惯的前提下适当拆分，避免中文句子“头重脚轻”。
- **禁止翻译的内容**：**论文标题（Title）、作者姓名（Authors）、所属机构（Affiliations）以及文末的参考文献（References）**必须保留外文原文，绝对不要将其翻译为中文。
- **段落一对一（防丢失警告）**：原文有几个段落，翻译后就必须有几个段落！绝对不允许擅自合并段落，绝对不允许遗漏任何一句话。请以逐句对照的态度进行翻译！

## 2. 强化学习与运控领域术语规范（核心）
- **中英对照原则**：对于核心概念，首次出现时建议采用“中文翻译 (English)”的格式，后续使用中文。例如：强化学习 (Reinforcement Learning), 域随机化 (Domain Randomization)。
- **保留英文不翻**：
  - **模型与算法缩写**：如 PPO, SAC, DDPG, Transformer, ResNet 等。
  - **常见基准或库**：如 MuJoCo, Isaac Gym, PyTorch 等。
  - **难以精准对应的行话**：如果中文翻译极度别扭（如 Vanilla, Ground Truth），可直接保留英文或使用行业黑话（原版、真值）。
- **标准术语映射表（参考）**：
  - State / Observation -> 状态 / 观测
  - Action / Policy -> 动作 / 策略
  - Reward / Penalty -> 奖励 / 惩罚
  - Sim2Real -> 仿真到现实（或直接用 Sim2Real）
  - Zero-shot / Few-shot -> 零样本 / 少样本
  - End-to-End -> 端到端
  - Robustness / Generalization -> 鲁棒性 / 泛化能力

## 3. 数学公式与符号（飞书兼容规范）
- **行内公式**：必须使用 `$math$`，**首尾绝对不允许有空格**。例如：正确 `$s_t$`，错误 `$ s_t $`。
- **块级独立公式**：必须使用 `$$` 且单独成行。禁止使用 `\begin{equation}` 等复杂环境，仅支持基础 KaTeX（如需换行对齐请用 `\begin{aligned}`）。
- **变量名称一致性**：正文中提到的单独字母变量（如“动作 $a_t$ 和奖励 $r_t$”）必须用 `$ $` 包裹，保持与公式中的字体一致。

## 4. 图表与排版映射
- **表格转换**：JSON 中的 Markdown 表格必须直接保留，并将表头及表格内的文字翻译为中文。**翻译时绝对不能破坏 Markdown 表格的管道符 `|` 对齐结构。**
- **图片与图注**：本流程不处理图片本身。如果正文中遇到对图表的文字说明（Caption，如 "Figure 1: ..."），请直接将其翻译为中文纯文本（如 "图 1：..."），以保证文本信息的完整性。
- ** 标题层级与连续性（严禁跳跃）**：翻译生成的 Markdown 标题层级必须严格连续递进，**绝对禁止跨层级跳跃**（例如绝不允许从 `# 一级标题` 下方直接出现 `### 三级标题`）。请遵循以下学术映射标
  - **一级大标题 `#`**：对应论文的顶级章节（如 `I. Introduction`, `II. Related Work`, `Abstract`）。
  - **二级小标题 `##`**：对应顶级章节下的子节（如 `A. Electromechanical Design`, `2.1 Background`），必须紧跟在一级标题下。
  - **三级小标题 `###`**：对应进一步的细分段落（如 `1)`, `a)` 等）。

## 5. 参考文献与引用（飞书兼容版）
- **禁止使用 HTML 锚点**：由于最终文档需上传至飞书，飞书不支持 `<span id="...">` 或 `#ref-1` 这样的 HTML 语法。**绝对禁止在正文或文末生成任何 HTML 标签。**
- **正文内引用处理**：正文中的参考文献数字请直接使用中括号包裹，并**加粗**以作视觉强调，方便读者在文档中按 `Ctrl+F` 搜索。
  - 英文：`...as shown by Vaswani et al. [1, 2].`
  - 翻译：`...正如 Vaswani 等人 **[1]**, **[2]** 所展示的。`
- **图表内链**：正文中提及图表（如 `see Fig. 1`），翻译为 `见图 1`，大小写应符合中文习惯。
- **文末参考文献列表**：按原样格式化为列表即可，无需添加任何特殊 ID 或链接符号。**必须保持外文原文，绝对禁止翻译参考文献的内容。** 例如：`[1] Vaswani, A., et al. (2017). Attention is all you need...`

## 6. 反例（禁止行为）
- **禁止输出 Markdown 包裹符**：因为你是按章节循环追加写入文件，输出的文本**绝对禁止**用 ` ```markdown ` 和 ` ``` ` 包裹，否则最终文件会出现大量冗余的代码块截断！直接输出纯文本形式的 Markdown。
- 禁止输出“这段话的意思是…”等作为 AI 助手的开场白。
- 禁止翻译代码块的内容（代码注释可酌情翻译，但代码本身和变量名绝不翻译）。
- 禁止在翻译中混入 AI 的个人评价或“启发”（评价只在生成论文笔记 `paper-note.md` 时做，翻译是纯客观的）。
- 翻译生成的 Markdown 标题层级必须严格连续递进， 绝对禁止跨层级跳跃。 （例如绝不允许从 `# 一级标题` 下方直接出现 `### 三级标题`）
- 禁止丢弃文末参考文献列表， 同时参考文献必须保持外文原文。
