# 上传预览展示规范

上传前必须把以下内容展示给用户确认。

## 目标

| 字段 | 说明 |
|---|---|
| `knowledge_base` | 目标知识库名称，来自配置 |
| `base` | 固定为 `论文索引` |
| `table` | 固定为 `数据表` |
| `view` | 固定为 `概览` |
| `note_parent_node_token` | `论文笔记` 父节点 token |
| `child_note_title` | 将创建或更新的论文子文档标题 |

## Base 字段

| 字段 | 来源 | 注意 |
|---|---|---|
| `Title` | 笔记元数据表 | 英文标题，建议 Title Case |
| `Abstract` | 笔记元数据表 | 英文摘要原文，不改写 |
| `标题` | 笔记元数据表 | `Title` 的忠实中文翻译 |
| `摘要` | 笔记元数据表 | `Abstract` 的忠实中文翻译 |
| `出版` | 笔记元数据表 | 会议、期刊或预印本来源 |
| `年份` | 笔记元数据表 | 出版年份 |
| `状态` | 固定 | 始终为 `待读` |
| `标签` | 笔记元数据表 | 数组形式 |
| `评分` | 笔记元数据表或用户确认 | 默认 `3`，可改为 `1-5` 或留空 |

## 附件

| 字段 | 说明 |
|---|---|
| `source` | 本地文件路径 |
| `upload_name` | 主 PDF 固定为 `paper.pdf` |

## 确认清单

- `Title`、`Abstract`、`标题`、`摘要` 严格来自笔记 `## 元数据` 表。
- `child_note_title` 不为空且不为 `Untitled`。
- 附件只有一个主文件 `paper.pdf`；附录按 `appendix.pdf`、`appendix-2.pdf` 命名。
- 新增 `标签`、`出版`、`年份` 选项已展示理由并获得确认。
- 若检测到同 Title 重复记录，已确认新增还是更新。
